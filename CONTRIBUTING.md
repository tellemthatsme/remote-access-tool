# Contributing to Remote Access Tool

Thank you for your interest in contributing!

## Development Setup

### Prerequisites

- Node.js 18+ 
- npm or yarn
- Git
- Windows 10/11 (for full functionality testing)

### Local Development

1. Clone the repository:
   ```bash
   git clone https://github.com/karma/remote-access-tool.git
   cd remote-access-tool
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Start the dashboard:
   ```bash
   npm start
   ```

4. Access at http://localhost:3001

## Code Style

- Use CommonJS (`.cjs` extension) for Node.js scripts
- Use descriptive variable and function names
- Add comments for complex logic
- Follow existing code patterns in the project

## Pull Request Process

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-feature`)
3. Commit your changes (`git commit -m 'Add amazing feature'`)
4. Push to the branch (`git push origin feature/amazing-feature`)
5. Open a Pull Request

## Reporting Bugs

Please include:
- Operating system version
- Node.js version
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

## Feature Requests

Open an issue with:
- Clear description of the feature
- Use case
- Potential implementation approach
